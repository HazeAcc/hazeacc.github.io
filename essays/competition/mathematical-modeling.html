# 数学建模参赛小记

# 初认识

第一次听说数学建模比赛是大一下学期快要放假的时候，那个时候离很多事情都很遥远，每天都在忙着两个专业的课程与两个社团部门的任务，在忙碌中体会充实感。偶然中听到同班的两个同学凑在一起说着美赛的事情，“数学建模”这个在当时听起来高大上的东西，对于连面对数学分析都在苦苦挣扎的我来说，我除了感慨一下身边的人真积极，心里对它还完全没有什么概念。

——后来遇到他们其中一个同学，问到关于数学建模的事情，他打哈哈道题目太难了，一点知识储备没有，挣扎了一个晚上就放弃了🤗

# 再听说

第二次听到“数学建模”的气氛则热烈得多了。大二与大三是第一次参加数学建模比赛的好时机，虽然按照知识储备来讲大三相对更加合适，但是身边的大二地同学们越来越频繁地提到诸如“数学建模”、“美赛”的字眼，充斥着躁动的气息，好似一股潮流，虽然大家都对其一知半解，但都生怕错过什么机会。这时候也有几个人来邀请我我一起组队，而我出于课程的繁忙犹豫之后还是推掉了。这可能与我不喜欢毫无储备就直接实战然后多层次递归学习的个性有关。

但说到底我们班作为数学与计算机双专业的实验班，和数学建模看起来不能再契合了。于是乎，随着大学时光的流逝，在18年美赛过后，我也开始尝试了解相关的内容，免得浪费了自己的专业优势。

# 深大杯

队友找了几个人很快就找齐了，虽然队友并不是大腿，都是没参加过数模比赛的萌新，但是对于数学建模的好奇仍然驱使着我们在闲暇之余偶尔了解数模相关的内容。

渐渐地，离19年国赛只剩下半个月了，学校安排为了筛选能够具备参加国赛资格的队伍，举办了一场国赛前的校内数学建模比赛——“深大杯”。其实我们都没有专门做系统的学习，进行深大杯的时候甚至我们都还在上课（所以比赛时间安排有整整5天）。拿到题的那个晚上我们讨论了很久选择哪道题目 —— 一道题是导弹发射的物理题，一道是调查中学教学改革的社会统计题。显然第一题较为基础而第二题较为发散，且从难度而言可以预见的是选择第一题的人数会很多，但作为新手，我们小组很乖地犯了新手该犯的错误，一直在纠结是否要尝试避开人群，选择第二题，并为此做出了许多尝试。

显而易见，这个新手常犯的错误使得我们直到第二天晚上才讨论出来结果，放弃第二题，着手第一题的建模。接下来的时间就是每天晚上大家都没课的时候聚在一起一点一点查找资料，规划导弹的运行轨迹。因为没有什么参赛准备，所以除了题目本身，甚至需要当场研究别人的论文怎么写、什么程度的简化假设还在合理范围内......

# 建模过程

当时的我们三个还没有明说谁是队长，两个队员一个是双修数学金融的同学A，比赛期间主要和我一起建模；另一个只是数学系的同学B，主要负责论文。但由于他们两个都没有学过大学物理，甚至因为没有写过实验报告，对于论文排版都很陌生，让我实在看不下去。于是乎，我不得不犯了一个“错误”：解题、建模、编程、论文校对等工作都交由我来承担。

在建模过程中，我与同学A一起解题，但他并没有做出什么实际性的帮助（因为的确不太擅长），使得我有了完整的解题思路后，还要花不少时间解释给A听，解决他的疑惑，然后他来进行那个部分的具体的建模。

论文的编写也问题很多，首先是格式的问题，即便使用Word文档编写论文，同学B的各种操作的了解还是有点少，我之好边建模边帮忙调整格式，教授如何使用Word的一些“高级操作”。

好在在第三天晚上，物理不好的同学B在查阅资料的时候找到了原题所对应的几篇优秀论文，我们一边怀揣着罪恶感一边饥渴地阅读优秀论文们，照猫画虎地进行自己的模型构建。在无尽地纠结中，我们一遍遍简化模型直到它能够简单地被物理公式表示并可计算，最后画出来的还是一条看起来十分简单实际上确实我们想破头去拟合的曲线哈哈😂

由于进度爆炸，最后一个晚上我们自然是需要通宵完成的，也是深大杯比赛过程中切切实实进度最快的一个晚上：这个晚上我们一下子完成了最后一半题目的模型构建，几乎肝完了3/4的论文，我尽可能地把模型给同学A解释清楚避免他建出来的模型又和我说的不一样，然后三个人分工写论文，期间顺便操起我不熟练的python画出各种需要插在文中的图片。结果自然导致我们写出了一篇糅合了三种不同风格的论文，而天也微微亮了，他们期间顶不住了小憩一会，于是我只好最后负责起论文的校对，统一文风、修改各种不规范的表达、整理格式，顺便赶路去上早上的课程（课程签到从来一个不落的乖孩子）。

# 提交论文

我一边上课一边完成了论文，将它交上给比赛举办方，然后做了每个提交论文的人都会做的“反复确认自己的提交是否有问题”的行为。一直十分精神的我终于将挺直的腰慢慢靠在了椅子上，一瞬间眼前恍惚，通宵的疲惫感忽然就像烟雾一样充满了我的大脑，然后是上半身，直到全身。仅几秒钟的时间，就好像要和椅子融为一体一般。

此时下课铃响，我又精神抖擞了一下。站起身，只感觉身体有点没有实感。回宿舍的路上的夏日阳光不那么热，来往的同学和往常一样又感觉有点那么一点不一样。开了空调，躺上床，感受脑袋里的血液充盈，在睡着前似乎体会到“数学建模”这四个字是怎样一种感觉。

# 赛后

最后结果出来，那篇在现在的我看来过于简单、论文格式糟糕、摘要不够优秀的论文也混了个三等奖，成功拿到参加国赛的许可，也算是差强人意吧。

我总结反思其实这次国赛前的预热，除了我们水平上、知识储备上不足以外，还有许多新手会犯的错误，实际上虽然另外两名队员的表现不算好，但是也确实有在出力，并不是队员都要非常的强才能进行比赛，关键还是分工协作的配合，以及细节的处理。况且这次比赛下来不得不说我也觉得自己很多方面也没有比两位队员强多少，也存在一些不如他们的地方，如同学A纯数学能力较强，同学B信息搜索能力较强，将这些优点互补便是数模团队合作的意义所在。

当然，通过这次比赛，他们都认可了我的能力，默认了我作为小队的队长😄。我也吸取这次比赛的教训，准备在国赛好好发挥各自的特长，尽可能发挥我们最好的水平。

# 过渡

说是既然初步筛选通过，就该在暑假好好准备国赛了，但实际上我们还是日常有些划水，毕竟数学建模的东西在大学学习生活中不太多用得到（虽然其实不少和计算机科学中的算法、数据处理等方面是一样的（其实就是懒）），指导老师也只开过两次会，人很好就是其他事情比较忙，因为我们学校并不算重视数学建模比赛（没有什么保研资格之类的奖励，只是兴趣），所以我们抱着玩一玩的心情准备参加一次国赛，反正三天时间换一次参赛经历一点也不亏嘛。

国赛开始时，正好是我大三刚开始的这个学期。经过大二的学习与指导老师基础的指导，对于如何写论文我已经有了大体的感觉了（也正是通过这些学习，愈发地觉得之前论文的拙劣= =）。开赛前几天好好地复习一下python与一些相关库（matplotlib、pandas、numpy）的用法（我用python代替matlab编程），又准备好一些资料以便于到时候查阅。指导老师贴心地为我们安排了单独的、不会受其他组影响的一个房间，这就是我们需要奋斗三天的地方。

# 国赛

# 第一天 选题

比赛是下午六点开始的，但是我晚上还有一门课，佛系的我们决定先各自看题，等我上完课再一起讨论题目。

六点到来的时候，我一边走路一边看题，打开队友发来的A题，好家伙，又是一道物理题，什么高压油管，什么膨胀压力，别说不会物理的队友们，我看了也直摇头——再做物理题是真的有点腻了，而且到最后肯定是演变为我一人出了绝大部分力的情况，物理也不是我的强项，果断放弃之，然后打开B题——怎么还是物理题？？

稍微看了一会，同心鼓🥁、击球角度、力学分析、误差分析......我有点汗颜，虽然感觉上要比A题要简单（注：后来发现选B的人最少，因为题目更难），仍然果断关闭，打开C题，终于看到一道我心目中比较像数学建模而不是物理建模（但其实是语文建模）的题目：

粗略了解题目意思后，又翻看了D题与E题，D题是一道典型的数据挖掘的题目，而且提供了数据不像C题需要找数据。就在我凭第一印象都想要选择D题时，队友告诉我D题与E题都是专科的题目，本科只能选择ABC，我：“......”

我们晚上八点半开始聚在一起，首先收拾了一下房间（继续佛系x），然后开始讨论问题。首先题A在不到2分钟的讨论时间内就被集体pass掉了（A题R.I.P.），然后开始考虑B题与C题，首先确认的是对于我们队，C题要比B题好做，但选B题的人数肯定比C少，我们又稍微纠结了一下是否要尝试B题，因为老师之前和我们说每个题获奖的队伍数是相对均衡的，选B题可能竞争没有那么激烈（但是网上又说是按照选题队伍数的比例分配的）。

与其空想纠结，不如实际动手。我们花了一个小时左右的时间尝试讨论B题的解题思路（其实主要是我在做，他们只得尝试跟着我的思路，或者搜索一些资料），后来越分析越发现一些连续的受力过程实在是超出了现有的知识储备，再加上我们队伍本身就不具备解决物理题的能力（所以说组队最好选择不同专业的队友比如计算机、地理测绘、物理等理工科专业更加吃香），在当晚结束前我们正式确定以C题为我们的比赛题目。

由于是两道物理题的情况，我们队伍确认题目的速度理论上可以更快，但实际上，花费半天的时间确认题目已经算快的，如果三道题都是队伍可以接受的题目，则一定要抓紧时间多多尝试才能尽早地定下题目（一天以内为佳），但也不能过快导致题目做一半做不下去出现换题的最坏情况。

接下来就是各自回去好好睡觉，约好第二天早上八点半开始做题咯。

# 第二天 理思路

一开始我们的策略是各自查阅一些资料，然后产生一些想法后开始讨论。头脑风暴少不得各种记录，思维导图是一个很好的方式。

有了一些想法后就是具体实现了，这一阶段我们队的分工是：我与同学A尝试解题，分享并统一思路，建立模型；同学B则专门收集资料，查找有价值的论文与相类似的往年国赛题目优秀论文。思路其实还算顺畅，但是没有数据的问题亟待解决。同学A从金融专业的角度提出一些专业名词，我听后认为可行，就按照他的方案对第一题进行建模。事实上经过一段时间的学习，大家的水平都有了不小的提高，相对而言我的数学能力还算有点拖后腿的，于是我也开始搜集数据了。

第二天慢慢过去了，我们也算逐渐进入比赛的状态之中，但建模的进度并不算快，一天结束，我们仅对题目进行了充分分析，收集到了一些与题目有关的论文，勉强完成了第一题的简单建模，顺便稍微思考了后面题目的思路，而比赛论文的编写、实际数据的收集还不见踪影。忙到了凌晨一点，我们统一了一下目前的进度与想法，便各自回寝室洗漱休息。

# 第三天 建模、数据、资料

第三天一大早，便开始接上第二天的思路进行建模，不断地尝试如何解决问题，做出合理的假设。但出于经验的不足，做出来的模型在我们自己看来有点简易，缺乏专业的依据，不由的心情稍微有点急躁。

为了查找有用的数据集，简单的翻遍百度显然并不能得到答案，但是找到了相关的大数据竞赛题目，只是由于时间已过期，赛方关闭了数据下载通道。于是我通过 github 搜索题目，尽管仅有项目搜索结果没能得到数据集，但是通过搜索commit信息与md文件信息，成功找到了一份数据集，是上海白云机场的半个月内客流量实时变化数据集。由此可见单靠搜索引擎搜不到想要的数据时，要从一些专业的网站入手才能找到不可能会被收索引擎收录的文件。
